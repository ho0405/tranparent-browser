# 투명 브라우저 배포 가이드

## 1. 설치 파일 만들기 (빌드)

프로젝트 폴더에서:

```bash
npm install
npm run build
```

빌드가 끝나면 **`dist`** 폴더에 설치 파일이 생성됩니다.

---

## 2. OS별 결과물

| 사용자 OS | 빌드하는 PC | 생성되는 파일 위치 |
|-----------|-------------|---------------------|
| **Windows** | Windows에서 빌드 | `dist/투명 브라우저 Setup 1.0.0.exe` (NSIS 설치 프로그램) |
| **macOS** | Mac에서 빌드 | `dist/투명 브라우저-1.0.0.dmg` |

- **Mac에서 빌드** → Mac용 `.dmg`만 생성  
- **Windows에서 빌드** → Windows용 `.exe`만 생성  
- 다른 OS용 설치 파일이 필요하면 **해당 OS가 있는 PC에서 각각 빌드**해야 합니다.

---

## 3. 사용자에게 전달하는 방법

1. **파일 공유**
   - `dist` 폴더 안의 **설치 파일 1개**만 전달 (예: `투명 브라우저-1.0.0.dmg` 또는 `투명 브라우저 Setup 1.0.0.exe`).
   - Google Drive, Dropbox, WeTransfer, GitHub Releases 등에 올려서 링크로 공유하면 됩니다.

2. **GitHub Releases (추천)**
   - GitHub 저장소에 코드를 올린 뒤, Releases에서 버전 태그를 만들고 `dist`에서 생성된 `.dmg` / `.exe`를 업로드하면, 사용자가 버전별로 다운로드하기 좋습니다.

3. **설치 방법 안내**
   - **Windows**: `투명 브라우저 Setup 1.0.0.exe` 실행 → 설치 경로 선택 → 설치 후 바탕화면/시작 메뉴에서 실행.
   - **macOS**: `투명 브라우저-1.0.0.dmg` 열기 → 앱 아이콘을 Applications 폴더로 드래그.

---

## 4. (선택) 코드 서명 없을 때 경고

- **macOS**: 개발자 인증서로 서명하지 않으면 “신뢰할 수 없는 개발자” 경고가 뜰 수 있습니다.  
  → **시스템 설정 → 개인 정보 보호 및 보안**에서 “그래도 열기”로 실행 가능하다고 안내하면 됩니다.
- **Windows**: 스마트스크린에서 “알 수 없는 게시자” 경고가 뜰 수 있습니다.  
  → “추가 정보” → “실행”으로 진행할 수 있다고 안내하면 됩니다.

코드 서명을 하려면 Apple Developer / Microsoft 인증서가 필요하고, `electron-builder` 설정에 서명 옵션을 추가해야 합니다.

---

## 5. 버전 올려서 재배포

새 버전을 배포할 때는 `package.json`의 `version`을 수정한 뒤 다시 빌드합니다.

```json
"version": "1.1.0"
```

```bash
npm run build
```

그다음 `dist`에 새로 생긴 설치 파일만 배포하면 됩니다.
